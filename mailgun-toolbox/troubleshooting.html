Dealing with a high bounce rate

A bounce rate above 5% can negatively impact your sender reputation and email deliverability. Here’s what to do if your bounce rate is climbing.

Common causes

  • Stale email lists — sending to addresses that no longer exist (people leave organizations, change providers).
  • Typos in email addresses — data entry errors in person records.
  • Purchased or imported lists — addresses that were never verified.
  • Domain changes — organizations that changed their email domain.

Steps to address a high bounce rate

  1. Review the Bounces tab — go to the Bounces tab and look at the addresses on the suppression list. Identify patterns (e.g., many addresses at one domain).
  2. Clean your people data — in Rock, find and update or inactivate person records with invalid email addresses. Use Rock’s Data Integrity features to find email duplicates and inactive addresses.
  3. Check the Details tab for error patterns — the Details tab shows the most common error codes. If you see 550 5.1.1 (user unknown) frequently, you have stale addresses.
  4. Monitor the Domains tab — if one domain has a high failure rate, investigate whether that domain is still valid.
  5. Remove known-bad addresses from Rock — mark email addresses as inactive in Rock for people who have permanently invalid addresses.
  6. Consider an email verification service — for large lists, use a third-party email verification tool to validate addresses before sending.

Mailgun automatically adds hard-bounced addresses to its suppression list, so they will not receive further emails. However, cleaning up the source data in Rock prevents wasted sends and improves your statistics.

Common issues

Problem Solution
“Mailgun API key is not configured” Set the Mailgun API Key global attribute. See Setup Guide.
“Mailgun domain is not configured” Set the Mailgun Domain global attribute. See Setup Guide.
Stats return zeros Verify the domain matches your Mailgun sending domain exactly.
Hourly resolution returns an error Mailgun limits hourly data to short ranges. The plugin clamps to 48 hours automatically. Try a shorter date range.
Complained count doesn’t match clickable list Stats count individual events. If an address complained multiple times, each event is counted separately.
Failed count seems high Only permanent failures are reported. Temporary failures (retries) are excluded from the count.
Dashboard page is blank or shows a loading spinner Clear your browser cache and reload. If the issue persists, check the Rock exception log for errors related to “ManagedMail” or “Mailgun.”

Getting support

If you’re experiencing an issue not covered above, you can get help from the DTS development team.

How to report an issue

  1. Go to the Mailgun Toolbox GitHub Issues page.
  2. Click New Issue.
  3. Describe the problem, including: what you expected to happen, what actually happened, your Rock version, and any error messages you see.
  4. If possible, include a screenshot of the issue.

GitHub Issues is the supported channel for Mailgun Toolbox support requests.
