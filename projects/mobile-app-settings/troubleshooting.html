Troubleshooting

Troubleshooting & Support

Solutions for common issues when using the Mobile App Settings plugin.

Common issues

Problem Solution
Block shows “Not Configured” Set the Site block setting or pass ?SiteId= as a URL parameter. See Setup Guide.
Block is not discovered after deploy Ensure rockplugin.json is configured as an embedded resource in the .csproj file. Restart Rock.
AppSetting Lava filter not available Restart Rock after deploying the DLL. The filter registers via IRockStartup on startup, not via migration.
Admin page not created after install Verify the parent page GUID 89DBB635-99A7-4ED5-AD61-019881279951 exists in your Rock instance. This is the default Rock “Installed Plugins” page.
Settings not saving Confirm the Site ID is resolving correctly. Check the block settings and ensure the site exists in Rock.
Categories from another site appear Each site has its own isolated category tree under a hidden root category (Mobile App Settings > Site_{SiteId}). Verify the correct Site ID is selected.
Lava filter returns empty string Check the setting key matches exactly (case-sensitive). If no site-specific value exists, the attribute’s default value is used. If no default is set, an empty string is returned.
Dashboard page is blank or shows a loading spinner Clear your browser cache and reload. If the issue persists, check the Rock exception log for errors related to “MobileAppSettings.”

Getting support

If you’re experiencing an issue not covered above, you can get help from the DTS development team.

How to report an issue

  1. Go to the Mobile App Settings GitHub Issues page.
  2. Click New Issue.
  3. Describe the problem, including: what you expected to happen, what actually happened, your Rock version, and any error messages you see.
  4. If possible, include a screenshot of the issue.

GitHub Issues is the supported channel for Mobile App Settings support requests.
