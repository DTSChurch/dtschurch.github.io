Mobile App Settings

Dynamic settings management for Rock RMS mobile apps

The Mobile App Settings plugin provides a hierarchical settings dashboard inside Rock RMS. Manage titles, hero images, feature flags, and any other configuration values for your mobile app — all without touching XAML or app code.

Get Started Setup Guide

What you get

  • Category tree: Organize settings into hierarchical categories with icons, descriptions, and drag-and-drop ordering.
  • Settings grid: View and edit settings by Name, Key, Type, and Value — all inline from the admin UI.
  • Any Rock field type: Text, Boolean, Date, Integer, and every other Rock field type is supported out of the box.
  • Per-site isolation: Each mobile app site manages its own independent settings tree.
  • Lava integration: Access any setting from Lava templates with the AppSetting filter — no migration required.
  • Lava example generator: Built-in modal generates copy-paste Lava snippets for every setting you create.

Settings are stored using Rock’s native Attribute system — no custom database tables required.

Requirements

  • Rock RMS v18.1+ (Obsidian)
  • .NET Framework 4.7.2
  • A mobile app site configured in Rock

Getting started

  1. Install the plugin — copy the DLL and Obsidian JS files into your Rock instance.
  2. Configure your site — set the Site ID and start creating categories and settings.
  3. Use settings in Lava — access your settings with the AppSetting filter.
