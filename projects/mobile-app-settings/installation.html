Installation

Installing Mobile App Settings

The Mobile App Settings plugin is deployed by copying two files into your Rock instance.

Install the plugin

Copy the following files from the plugin/ directory into your Rock installation:

Source Destination
plugin/Bin/com.dtschurch.MobileAppSettings.dll RockWeb/Bin/
plugin/Plugins/com_dtschurch/MobileAppSettings/mobileAppSettings.obs.js RockWeb/Plugins/com_dtschurch/MobileAppSettings/

Pre-built files are included in the plugin/ directory of the repository. If you need to build from source, see the Technical Guides.

Restart Rock

After copying the files, restart Rock (or recycle the application pool) to load the assembly and run the migrations.

What the plugin creates

On first startup the migrations will automatically:

  1. Register the Mobile App Settings entity type and block type under DTSChurch > Mobile.
  2. Create a Mobile App Settings admin page with the block already added.
  3. Register the AppSetting Lava filter globally via IRockStartup.

After restart, the admin page will be available in your Rock instance.

Advanced: Building from source

If you need to build the plugin yourself:

# C# Backend
dotnet build com.dtschurch.MobileAppSettings/com.dtschurch.MobileAppSettings.csproj -c Release

# Obsidian Frontend
cd com.dtschurch.MobileAppSettings.Obsidian
npm install
npm run build

Then copy the build outputs to your Rock instance:

Build Output Destination
bin/Release/net472/com.dtschurch.MobileAppSettings.dll RockWeb/Bin/
dist/mobileAppSettings.obs.js RockWeb/Plugins/com_dtschurch/MobileAppSettings/

Requires .NET 4.7.2 SDK, Node.js 18+, and Rock RMS v18.1 source at ../RockV18.1/RockWeb/.

Next step

Once installed, head to the Setup Guide to configure the block and start creating settings.
