Book Appointment Block

The Book Appointment block is the external-facing booking experience. It guides users through a multi-step flow: selecting an appointment type, choosing a date and time, entering contact information, and confirming the booking.

Multi-Step Flow

The block walks users through up to five steps depending on context:

Step 1: Appointment Type Selection

[Placeholder: Screenshot of appointment type cards with filters]

  • Displays appointment types as a responsive card grid
  • Each card is rendered via the Appointment Type Card Template (Lava)
  • Cards show availability count and are disabled when no slots exist
  • Optional campus and location filter dropdowns (enabled via block settings)
  • Skipped entirely when the appointment type is predetermined via block attribute or URL parameter

Step 2: Date & Time Selection

[Placeholder: Screenshot of date accordion with slot pills]

  • Available dates grouped in an accordion (first date expanded by default)
  • Time slots rendered via the Slot Card Template (Lava)
  • Slots show availability with color-coded indicators (scarce, limited, open)
  • Respects booking threshold (hours before appointment that changes are blocked)
  • Supports slot subdivision (e.g., a 2-hour schedule divided into 30-minute slots)

Step 3: Person Information

[Placeholder: Screenshot of person information form]

  • First name, last name, email (required), phone (configurable)
  • Auto-fills from the current logged-in person
  • Displays appointment-type-specific attributes (configured per type)
  • Optional notes field
  • Person matching uses Rock's built-in FindPerson logic

Step 4: Success Confirmation

[Placeholder: Screenshot of success confirmation]

  • Rendered via the Confirmation Template (Lava)
  • Confirmation email sent automatically if configured on the appointment type

Step 5: Cancellation (when applicable)

  • Shown when the user navigates with ?Cancel=true and a valid appointment GUID
  • Optional cancellation reason text field
  • Only available if the appointment type allows cancellation
  • Sends cancellation notification email and launches rescheduled workflow

URL Parameters

Parameter Purpose
AppointmentTypeId Pre-selects an appointment type (supports both integer ID and IdKey). Skips step 1.
Appointment GUID of an existing appointment to edit or reschedule.
Cancel Set to true to show the cancellation flow for the specified appointment.
CampusId Pre-filters appointment types to a specific campus.
Category Filters appointment types by category name.
ConnectionRequestId Associates the booked appointment with a Rock Connection Request.

Entity Reference Conventions

The block follows Rock's standard Obsidian conventions for entity references. No raw integer IDs are exposed to the client:

  • ListItemBag (Value = Guid, Text = name) for cross-entity references in display bags
  • Guid for entity references in save operations and block action parameters
  • IdKey only for the entity's own identity in card bags
