Setup Guide

Configuration & Dashboard

Connect the plugin to your Mailgun account and learn what each dashboard tab shows.

Configuration

1. Verify your Mailgun email transport

Before configuring the plugin, make sure Rock is already set up to send email through Mailgun:

  1. Go to Admin Tools > Communications > Communication Transports.
  2. Confirm that Mailgun HTTP is listed and marked as active.
  3. If Mailgun is not your active transport, configure it first per Rock’s documentation.

The Mailgun Toolbox reads stats from the same Mailgun domain that Rock uses to send email. If Rock is not configured to send through Mailgun, the dashboard will have no data to display.

2. Set the Mailgun API Key

  1. Go to Admin Tools > General Settings > Global Attributes.
  2. Click Add to create a new attribute with the key MailgunApiKey.
  3. Enter your Mailgun private API key (starts with key-... or is a hex string) as the value.
  4. Click Save.

3. Set the Mailgun Domain

  1. In the same Global Attributes page, click Add to create a new attribute with the key MailgunDomain.
  2. Enter the sending domain configured in Mailgun (e.g. mg.yourchurch.com) as the value.
  3. Click Save.

Verify it’s working

  1. Navigate to Admin Tools > Installed Plugins > Mailgun Toolbox.
  2. You should see the Overview tab with delivery rate cards and charts.
  3. If you see data for the current month, the plugin is correctly configured.
  4. If you see zeros or an error message, see the Troubleshooting page.

Using the dashboard

The dashboard gives you a complete view of your email health. A date range dropdown at the top applies to all tabs — choose from preset ranges or select Custom to enter your own dates.

Screenshot: Date range filter dropdown Replace this with a screenshot showing the date range selector.

Overview tab

Your at-a-glance email health summary:

  • Rate cards — delivery rate, open rate, click rate, and bounce rate as percentages, plus total counts.
  • Daily trend charts — line charts showing email counts, failure rate, and engagement rates over the selected period.
  • Complaint count — click the complained count to see individual complaint events.

What to look for

  • A healthy delivery rate is typically above 95%.
  • A rising failure rate trend may indicate a deliverability problem.
  • Low open or click rates may indicate content or list quality issues.

Details tab

Investigate delivery failures:

  • Failure severity breakdown — permanent failures (messages that could not be delivered) vs. temporary failures (retries that eventually succeeded or were abandoned).
  • Top error codes — a ranked list of the most common error codes with their frequency.
  • Recent failed events — the 25 most recent failed deliveries showing recipient, error message, and timestamp.

What to look for

  • Common error codes like 550 5.1.1 indicate invalid recipient addresses.
  • A spike in temporary failures may indicate a temporary issue with a receiving mail server.

Bounces tab

Email addresses on the Mailgun suppression list — addresses that previously bounced and will not receive future emails.

  • Search — use the search box to find a specific email address.
  • Delete — remove an address from the suppression list to allow Mailgun to attempt delivery again.
  • Pagination — browse through the full list page by page.

When to delete a bounce

  • If you know the recipient’s mailbox issue has been fixed (e.g., they cleared space or their IT team resolved a server issue).
  • If the bounce was caused by a temporary problem that has been resolved.
  • Caution: repeatedly sending to addresses that bounce can harm your sender reputation.

Historicals tab

Email volume and outcomes as a time-series table at three resolutions:

  • Hourly — best for investigating recent issues (limited to 48-hour windows).
  • Daily — good for week-over-week comparisons.
  • Monthly — useful for long-term trend analysis.

Click any non-zero failed count to drill into the specific failure events for that time period.

Domains tab

Per-recipient-domain breakdown (e.g., gmail.com, outlook.com, yahoo.com):

  • Total sent count
  • Delivered count
  • Hard failure count (permanent bounces)
  • Soft failure count (temporary issues)
  • Success rate percentage

What to look for

  • If a specific domain has a significantly lower success rate, there may be a deliverability issue with that provider.
  • Gmail and Outlook have stricter sending requirements — monitor these closely.
