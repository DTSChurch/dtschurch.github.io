Technical Guides

How the plugin is built

A look under the hood at the Mailgun Toolbox plugin architecture, block actions, and API integration. This page is for the technically curious — you don’t need any of this to use the plugin.

Project structure

com.dtschurch.ManagedMail/            # C# plugin (.NET Framework 4.7.2)
  Blocks/Mail/                        #   MailgunDomainStats block type
  Integration/                        #   Mailgun API v3 client & models
  Migrations/                         #   Rock database migrations (5)
  Constants/                          #   GUIDs and plugin metadata

com.dtschurch.ManagedMail.Obsidian/   # Vue 3 frontend (Obsidian)
  src/mailgunDomainStats.obs          #   Single-file component (~61 KB)

The plugin has two main parts: a C# backend that runs inside Rock and talks to the Mailgun API, and a Vue 3 frontend component that renders the dashboard in Rock’s Obsidian shell.

Block actions

The Obsidian frontend calls these block actions on the C# backend to fetch data and manage suppression lists:

Action Purpose
GetStats Aggregate statistics for the selected date range.
GetTimeSeriesStats Time-series data at hour/day/month resolution.
GetBounces / GetBouncesPage Paginated bounce suppression list.
DeleteBounce Remove an address from the Mailgun bounce suppression list.
SearchBounce Look up a specific bounce address.
GetEventsPage Paginated event log (failures, complaints, deliveries).
GetComplaints / DeleteComplaint View and manage the complaint suppression list.
GetDomainBreakdown Per-recipient-domain delivery statistics.

Mailgun API endpoints used

The C# backend proxies requests to these Mailgun API v3 endpoints:

Method Endpoint Purpose
GET /v3/{domain}/stats/total Aggregate and time-series statistics
GET /v3/{domain}/bounces Bounce suppression list
DELETE /v3/{domain}/bounces/{address} Remove a bounce
GET /v3/{domain}/events Event log (failures, complaints)
GET /v3/{domain}/complaints Complaint suppression list
DELETE /v3/{domain}/complaints/{address} Remove a complaint

Key source files

  • com.dtschurch.ManagedMail/Blocks/Mail/MailgunDomainStats.cs — the Rock block that handles all block actions and API calls.
  • com.dtschurch.ManagedMail/Integration/MailgunApiClient.cs — HTTP client wrapper for the Mailgun API.
  • com.dtschurch.ManagedMail.Obsidian/src/mailgunDomainStats.obs — the Vue 3 single-file component that renders the entire dashboard UI.
  • com.dtschurch.ManagedMail/Migrations/ — five Rock migrations that set up the block type, global attributes, and default page.

Building from source

# C# Backend
dotnet build com.dtschurch.ManagedMail/com.dtschurch.ManagedMail.csproj -c Release

# Obsidian Frontend
cd com.dtschurch.ManagedMail.Obsidian
npm install
npm run build

# Package as .plugin file
dotnet tool install --global sparkdevnetwork.rock.devtool
rock-dev-tool plugin pack
