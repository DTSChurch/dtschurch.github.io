Planning Center Settings

Single-tenant configuration block. Persists API credentials and sync policy to the _com_dts_PlanningCenter_Settings table (singleton row). Also exposes campus mapping shortcuts and manual job control.

Block Attributes

This block has no configurable block attributes — configuration lives in the singleton settings record.

What It Manages

FieldDescription
Application ID Planning Center API Application ID. Stored in plain text in ApiApplicationId.
Secret PCO API Secret. Encrypted at rest via Rock.Security.Encryption before write; decrypted on read for API calls.
Overwrite With Planning Center When on, PCO data overwrites Rock fields on existing linked records. When off, only missing fields are filled in. Default true in the initial form, false in the EF default.
Default Connection Status Defined Value GUID from PERSON_CONNECTION_STATUS. Applied to new people created during sync.

Block Actions

ActionBehavior
SaveSettings Encrypts the secret, upserts the singleton settings row, and resolves the connection status GUID to DefaultConnectionStatusValueId.
TestConnection Issues a GET people/v2 against api.planningcenteronline.com using HTTP Basic auth with the saved credentials. Returns Connected on 2xx, otherwise the HTTP status and reason.
RunSyncNow Looks up the SyncPlanningCenter service job and enqueues a RunJobNowTransaction. Refuses if LastStatus = "Running".
RefreshSyncStatus Returns LastRunDateTime, LastStatus, LastStatusMessage, and LastRunDurationSeconds from the service job for status polling.
GetCampusMappings Lists existing EntityMap rows where PlanningCenterEntityType = "Campus", joined to the Rock Campus for display.
GetPcoCampuses GET people/v2/campuses against PCO. Returns ID + name as ListItemBags.
GetRockCampuses Returns active Rock campuses from CampusCache.
SaveCampusMapping Upserts an EntityMap row of type Campus linking a PCO campus ID to a Rock campus ID. Sets SyncStatus = "Mapped".
DeleteCampusMapping Removes an existing campus EntityMap row by IdKey.

Planning Center Entity Mapping

Unified mapping editor for People, Groups, and Service Teams. The same block is placed on both the Groups and People admin pages; the Entity Type Filter attribute is what scopes each instance.

Block Attributes

AttributeTypeDescription
Entity Type Filter EntityType Optional. Restricts the block to a single Rock entity type (e.g. Group for the Groups page, PersonAlias for the People page). Leave blank to show every mapping type.
Person Detail Page LinkedPage Optional. Page to navigate to when a person row is clicked. Typically the Person Profile.
Group Detail Page LinkedPage Optional. Page to navigate to when a group row is clicked. Typically the Group Viewer.

Mapping Types It Handles

  • Group — Rock Group ↔ PCO Group.
  • Team — Rock Group ↔ PCO Service Team. Supports per-team-position role mapping into Rock Group Roles, persisted as JSON in EntityMap.AdditionalData.
  • Person — Rock PersonAlias ↔ PCO Person. Created automatically by the sync; surfaced here for inspection and manual cleanup.

Caching

PCO Service Types and Teams are cached in-memory under PlanningCenter_ServiceTypesWithTeams for one hour to avoid hammering the PCO API during configuration. The cache is refreshed automatically when the entry expires.

Planning Center Entity Link

Inline display block. Reads the page’s Person or Group context, looks up an active EntityMap entry, and renders a deep link into Planning Center if one is found. Returns isVisible = false (renders nothing) when no mapping exists, so it is safe to place on any detail page.

Page Parameters Recognized

ParamResolves To
PersonId / PersonGuid / PersonRock Person (the Person param accepts IdKey, GUID, or integer ID).
GroupId / GroupGuid / GroupRock Group (same fallback chain).

Deep Link Targets

PCO Entity TypeURL Pattern
Person https://people.planningcenteronline.com/people/{pcoId}
Team https://services.planningcenteronline.com/teams/{pcoId}
Group https://groups.planningcenteronline.com/groups/{pcoId}/members?...

Person Mapping Note

Person mappings are stored against PersonAlias, not Person. The block resolves the current Person to all of their alias IDs and looks up the EntityMap by the alias set, so a Person who has been merged in Rock still resolves correctly.

Block Actions

ActionBehavior
SyncGroup Synchronously runs PlanningCenterSyncService.SyncServiceTeam or SyncPlanningCenterGroup for the single mapping shown on the page. Updates SyncStatus on the EntityMap and returns add/update counts. Used by the “Sync Now” affordance on the Group Viewer.

Planning Center Sync History

Read-only grid of past SyncLog rows ordered by StartDateTime descending. Each row shows the sync phase, start/end time, computed duration, status, and per-bucket counts (processed / created / updated / skipped / errored). A detail view exposes the full ErrorDetails blob.

Block Attributes

None. The block reads directly from the _com_dts_RockLinkPlanningCenter_SyncLog table.

Block Actions

ActionBehavior
GetGridData Returns every SyncLog row, projected to SyncLogBag with computed DurationSeconds.
GetSyncLogDetails Returns a single SyncLog row by IdKey for the detail panel.
