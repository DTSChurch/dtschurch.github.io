Blocks

The plugin ships four admin blocks under the DTS > Planning Center category. All four are installed and placed on their default pages automatically — you only need to add a block yourself if you want the Entity Link to appear on a custom detail page.

Block Where It’s Installed
Planning Center Settings RockLink › Settings
Planning Center Entity Mapping RockLink › Groups, RockLink › People
Planning Center Entity Link Person Profile, Group Viewer
Planning Center Sync History Available; not auto-placed in v1

Planning Center Settings

Central configuration block. This is where you connect Rock to Planning Center, decide how conflicts are resolved, set the default Connection Status for new people, manage your campus mappings, and trigger a sync on demand.

What you configure

FieldDescription
Application ID Your Planning Center API Application ID.
Secret Your Planning Center API Secret. Encrypted before it’s saved — once entered, it is not displayed again in plain text.
Overwrite With Planning Center When on, PCO data overwrites the corresponding Rock fields on linked records. When off, Rock data is preserved and only missing fields get filled in.
Default Connection Status Connection Status assigned to brand-new people created by the sync.

Actions on this block

ButtonWhat it does
Save Stores your credentials and sync policy.
Test Connection Reaches out to Planning Center with the saved credentials and reports whether the connection succeeded.
Run Sync Now Queues the Planning Center sync job to run right away. Disabled if a sync is already in progress.
Job Schedule Shortcut into the Rock Job Scheduler to view or change the sync’s schedule.

Last Sync Status panel

The block surfaces the most recent run’s date, status, status message, and duration. After clicking Run Sync Now, this panel refreshes automatically so you can watch progress without leaving the page.

Campus mapping

The block also includes a campus mapping editor. It pulls the list of campuses from Planning Center, lets you pick the matching Rock campus for each, and saves the link. You can edit or remove a campus mapping here at any time.

Planning Center Entity Mapping

The mapping editor for People, Groups, and Service Teams. The same block lives on both the Groups and People admin pages; each instance is scoped to the type of mapping that page is responsible for.

Block settings

SettingDescription
Entity Type Filter Optional. Restricts the block to a single type of mapping (e.g. only Group mappings on the Groups page). Leave blank to show every type.
Person Detail Page Optional. The page to navigate to when a person row is clicked — typically the Person Profile.
Group Detail Page Optional. The page to navigate to when a group row is clicked — typically the Group Viewer.

Mapping types it handles

  • Group — Rock Group ↔ Planning Center Group.
  • Service Team — Rock Group ↔ Planning Center Service Team. Supports mapping each PCO team position to a Rock group role.
  • Person — Rock Person ↔ Planning Center Person. Created automatically by the sync; surfaced here so you can review or clean up links.

What you can do here

  • Create a new mapping by picking a Rock Group and the matching PCO Group or Service Team.
  • For Service Team mappings, configure which PCO team position becomes which Rock group role.
  • Activate or deactivate a mapping. Inactive mappings are skipped by the sync without losing the link.
  • Remove a mapping you no longer want synced.

Planning Center service types and teams are cached for an hour to keep the editor responsive — if you don’t see a brand-new team yet, wait a few minutes and reload.

Planning Center Entity Link

A small inline block that appears on Person Profile and Group Viewer pages. When the record being viewed is linked to a Planning Center counterpart, the block shows a Planning Center logo and a deep link straight to the matching record in PCO. When there’s no link for the current record, the block stays hidden, so it’s safe to leave on every detail page.

What it links to

Linked RecordOpens
PersonThe matching person in Planning Center People.
Service TeamThe matching team in Planning Center Services.
GroupThe matching group’s member list in Planning Center Groups.

Sync Now (Group Viewer only)

On the Group Viewer, the block also includes a Sync Now button. Clicking it runs the sync for just that one group right away — useful when you’ve made a change in Planning Center and want it reflected in Rock without waiting for the next scheduled run. Add and update counts are reported inline when the run finishes.

Planning Center Sync History

A read-only grid of every past sync run, newest first. Use it to confirm a sync actually ran, see how many records were touched, and dig into errors when something looks off.

Columns shown

  • Sync phase
  • Start & end date/time, plus computed duration
  • Status — Running, Completed, Completed With Errors, or Failed
  • Records processed, created, updated, skipped, and errored
  • Error details (in the row detail view)

The grid is the easiest place to confirm at a glance whether the most recent run finished cleanly. Clicking a row opens a detail view with the full error message, when applicable.
