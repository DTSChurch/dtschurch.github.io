The Planning Center Sync Job

A single Rock Job drives the sync. It uses the credentials and policy you saved on the Settings page, walks every active mapping you’ve set up, and writes a history record for every run. You can see the job in Admin Tools → System Settings → Jobs Administration, or follow the “Job Schedule” shortcut from the Settings block.

Configuration

Schedule

Like any Rock Job, the schedule is configured in the Job Scheduler — pick whatever cadence fits your data. Hourly is a reasonable starting point for active tenants. The job can run as often or as infrequently as you like; each run fully reconciles every active mapping.

Job settings

SettingDefaultDescription
Command Timeout 180 seconds Maximum time any single database operation in the sync may take before timing out. Increase this if you have a very large mapping that consistently exceeds the default.

What it reads from Settings

  • Your Planning Center API credentials.
  • The default Connection Status applied to brand-new people.
  • The overwrite policy — whether PCO data overwrites Rock fields or only fills in blanks.

If credentials haven’t been entered yet, the job stops cleanly before calling Planning Center, records the reason in the run history, and reports the failure in the Jobs grid.

What Happens On Each Run

  1. The job starts and writes a new Running entry into the sync history.
  2. It loads your saved settings and looks up every active group/team mapping you’ve configured.
  3. For each mapping, it pulls the latest data from Planning Center — the group or team itself, plus its members.
  4. For each member, it finds or creates the matching Rock person, then adds or updates that person’s membership in the linked Rock group (with the right group role for Service Teams).
  5. Per-mapping totals are rolled up into run-level counts and the Jobs grid status message is updated as it goes, so you can watch progress.
  6. When everything is done, the run history entry is updated with the final counts and a status of Completed, Completed With Errors, or Failed.

What Each Mapping Does

PCO Group

Group sync

Pulls the group from Planning Center Groups, syncs every member as a Person (creating or updating per the overwrite policy), then adds or updates each membership in the linked Rock Group.

PCO Service Team

Service team sync

Pulls the team and its members from Planning Center Services, syncs each member as a Person, then adds or updates each membership in the linked Rock Group. PCO team-position → Rock group-role mapping is applied if you’ve configured one.

Across all mappings

Matching people

If a person is already linked to PCO, that link is used. Otherwise the sync looks for a matching Rock person by name, email, and phone, links them on first match, and creates a brand-new record only if no match is found.

Error Handling

  • One bad record doesn’t stop the run. Per-record errors are recorded onto the relevant counter and the run keeps going.
  • The first 10 errors are echoed into the job’s status message so they show up at a glance in the Jobs grid. Up to 50 are saved on the run history record for later review.
  • If a whole mapping fails — for example, the matching team was deleted in Planning Center — that mapping is flagged with an Error status and the error message is stored on it. Other mappings still process normally.
  • If the run itself can’t start (e.g. credentials missing, Planning Center is unreachable), the run history entry is marked Failed with the reason, and the failure shows up in the Rock Jobs grid.
  • End-of-run status is Completed With Errors if anything was caught along the way; otherwise Completed.

Running Sync On Demand

You don’t have to wait for the schedule. There are two ways to trigger a sync immediately:

  • Run Sync Now on the Settings page — queues a full run of every active mapping. The button is disabled if a sync is already in progress.
  • Sync Now on the Group Viewer (via the Entity Link block) — runs the sync for just that one group. Useful for verifying a single change without re-running everything.

What This Job Is Not Doing

  • It only processes Rock Groups linked to PCO Groups or Service Teams. Person and Campus mappings are created and consumed during a run, but they aren’t walked as a separate phase.
  • There is no incremental-only mode in v1 — every active mapping is fully reconciled on every run.
  • It does not pull attendance or volunteer schedules.
  • It does not push anything from Rock back to Planning Center.
