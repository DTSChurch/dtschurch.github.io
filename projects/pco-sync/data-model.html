Schema

Data Model

The plugin adds three custom tables to the Rock database. All three live in the dbo schema with the _com_dts_ prefix and are tagged with the Rock domain DTS > Planning Center.

TablePurposeCardinality
_com_dts_RockLinkPlanningCenter_EntityMap Links between PCO entities and Rock entities (Person, Group, Campus, GroupMember). One row per linked entity.
_com_dts_RockLinkPlanningCenter_SyncLog Run history for the sync job — counts, status, duration, errors. One row per sync run (or partial / failed run).
_com_dts_PlanningCenter_Settings Singleton settings record — API credentials and sync policy. Exactly one row.

EntityMap

The single linking table. One EntityMap row represents one “this PCO thing is the same as that Rock thing” relationship, scoped by the Rock EntityType on the Rock side and a string PlanningCenterEntityType on the PCO side.

Columns

ColumnTypeDescription
PlanningCenterEntityTypenvarchar(50), requiredOne of Person, Group, Team, Campus, GroupMember.
PlanningCenterEntityIdnvarchar(50), requiredThe PCO-side identifier.
PlanningCenterNamenvarchar(200)Cached display name from PCO so the UI does not have to refetch.
PlanningCenterModifiedDateTimedatetime, nullableLast-modified timestamp pulled from PCO. Reserved for future delta-sync logic.
EntityTypeIdint, required, FK → EntityTypeThe Rock entity type the link resolves to. For Person mappings this is PersonAlias, not Person.
EntityIdint, requiredThe primary key of the linked Rock entity.
ParentEntityMapIdint, nullable, FK → EntityMap.IdFor GroupMember rows, points to the parent Group / Team mapping. Self-referential.
AdditionalDatanvarchar(max)Free-form JSON. Used today for PCO team-position → Rock GroupRole mappings on Service Team rows.
LastSyncDateTimedatetime, nullableMost recent successful sync of this mapping.
SyncStatusnvarchar(20)One of Synced, Mapped, Syncing, Error, Pending.
ErrorMessagenvarchar(max)Last error if SyncStatus = "Error". Cleared on the next successful run.
IsActivebit, requiredThe sync job only iterates rows where IsActive = 1. Default 1.
Plus the standard Rock entity columns: Id, Guid, CreatedDateTime, ModifiedDateTime, CreatedByPersonAliasId, ModifiedByPersonAliasId, ForeignKey, ForeignGuid, ForeignId.

Relationships

  • EntityType — required FK, no cascade delete. Identifies which Rock entity type the link resolves to.
  • ParentEntityMap — optional self-FK, no cascade delete. GroupMember rows point at their owning Group / Team mapping; deleting a parent does not cascade to child rows in v1.

Why Person mappings target PersonAlias

Person rows in Rock can be merged. Merging rewrites the surviving Person’s primary alias but preserves the merged-away aliases pointing at the surviving record. By storing the link against the PersonAlias, a PCO Person stays correctly resolved to the surviving Rock Person after a merge without any sync-side cleanup.

Conventions

  • Group ↔ PCO Group: EntityTypeId = Group, PlanningCenterEntityType = "Group".
  • Group ↔ PCO Service Team: EntityTypeId = Group, PlanningCenterEntityType = "Team".
  • Campus ↔ PCO Campus: EntityTypeId = Campus, PlanningCenterEntityType = "Campus".
  • PersonAlias ↔ PCO Person: EntityTypeId = PersonAlias, PlanningCenterEntityType = "Person".

SyncLog

One row per sync run. Created when the job starts (with Status = "Running") and updated to a terminal status when the run ends. Powers the Sync History block.

Columns

ColumnTypeDescription
JobIdint, nullableThe Rock ServiceJob.Id that triggered the run, when applicable.
SyncPhasenvarchar(50), requiredOne of People, Groups, GroupMembers, All. v1 always writes "All".
StartDateTimedatetime, requiredWhen the run started.
EndDateTimedatetime, nullableWhen the run ended. NULL while in progress.
RecordsProcessedintCreated + updated + skipped, summed across People, Groups, and Members.
RecordsCreatedintTotal new rows created across all three buckets.
RecordsUpdatedintTotal existing rows updated across all three buckets.
RecordsSkippedintTotal rows skipped across all three buckets.
RecordsErroredintTotal per-record errors across all three buckets.
Statusnvarchar(20), requiredRunning, Completed, CompletedWithErrors, or Failed.
ErrorDetailsnvarchar(max)Up to the first 50 per-record error messages joined by newline, or the fatal exception message for Failed runs.
Plus the standard Rock entity columns.

DurationSeconds is not stored; it is computed in the block layer as EndDateTime - StartDateTime.

PlanningCenterSettings

Singleton table. The block UI enforces a single row by always upserting the first existing record. There is no schema-level constraint preventing additional rows — readers consistently call .FirstOrDefault().

Columns

ColumnTypeDescription
ApiApplicationIdnvarchar(100), requiredPCO Application ID. Stored in plain text.
ApiSecretnvarchar(500), requiredPCO Secret — encrypted via Rock.Security.Encryption before save. Decrypted only at API-call time.
DefaultConnectionStatusValueIdint, nullable, FK → DefinedValueConnection Status applied to brand-new Person rows created by the sync.
OverwriteWithPlanningCenterbitEF default is 0 (false). The Obsidian block defaults a brand-new form to true, so a freshly configured tenant lands at “PCO wins.”
Plus the standard Rock entity columns.
