Planning Center Sync

One-way sync from Planning Center Online into Rock RMS

Planning Center Sync pulls People, Groups, Service Teams, and Campuses from Planning Center Online (PCO) into Rock on a configurable schedule. PCO is treated as the source of truth; Rock is the destination. You manage the connection, mappings, conflict policy, and run history from a small set of admin pages installed under Admin Tools → RockLink.

Custom Plugin Rock RMS v17.2+ Scheduled Job

What It Syncs

People

Name, email, phone, address, birthdate, gender, photo, and campus. New people are created with the Connection Status you choose; existing people are matched to their Planning Center counterpart and kept in step.

Groups & Service Teams

Rock Group ↔ PCO Group or PCO Service Team, including group members. For Service Teams, you can map each PCO team position to a Rock group role.

Campuses

Rock Campus ↔ PCO Campus. Used to assign synced people and groups to the correct Rock campus.

What It Does Not Sync (v1)

  • Attendance records — not pulled from PCO.
  • Volunteer schedules — scheduled assignments (e.g. “Person X serving on Date Y”) are not imported.
  • Bi-directional sync — Rock does not push changes back to PCO. The flow is PCO → Rock only.

These are on the future-enhancements list and are not part of v1.

How It Works

1. Configure the connection

Enter your Planning Center API Application ID and Secret in the Settings page. The secret is encrypted before it’s saved. Use Test Connection to verify the credentials work.

2. Map your data

On the Groups page, link each Rock Group to the matching PCO Group or Service Team. On the Settings page, link each Rock Campus to the matching PCO Campus. For Service Teams, optionally map each team position to a Rock group role.

3. Let the sync run

The Planning Center sync job runs on a schedule you configure (or on demand via Run Sync Now). It walks every active mapping, fetches the latest data from PCO, and creates or updates the matching Rock records based on your overwrite policy.

4. Review history

The Sync History view shows every run with totals (added / updated / skipped / errored), duration, status, and any error details for diagnostics.

Key Configuration Knobs

Setting Effect
Overwrite With Planning Center When on, PCO data overwrites the corresponding fields on linked Rock records. When off, Rock data is preserved and only missing fields get filled in.
Default Connection Status The Connection Status assigned to brand-new people created by the sync.
Active mappings Only mappings flagged as active are processed. Deactivating a mapping pauses sync for that group/team without removing the link.

Behavior worth knowing

  • Once a person has been linked to a Planning Center record, the sync preserves their Rock name on later runs — Rock is treated as authoritative for the name from that point on. Other fields still update per the overwrite policy.
  • The very first time a person is matched to PCO (by name, email, or phone), the PCO name is applied — that’s how a brand-new link gets seeded.
  • If a Rock person is later merged with another, the Planning Center link follows the surviving record automatically — no cleanup needed.

Admin Pages Installed

Installing the plugin adds a new RockLink section under Admin Tools, with three child pages, plus surfaces on the Person Profile and Group Viewer.

Page What’s On It Purpose
RockLink › Settings Planning Center Settings block API credentials, sync policy, campus mapping, manual run trigger.
RockLink › Groups Planning Center Entity Mapping block Manage Rock Group ↔ PCO Group / Service Team mappings.
RockLink › People Planning Center Entity Mapping block Inspect Rock Person ↔ PCO Person mappings.
Person Profile Planning Center Entity Link block Adds a “View in Planning Center” link on a person’s profile when a mapping exists.
Group Viewer Planning Center Entity Link block Adds a deep link into Planning Center and a per-group Sync Now button when a mapping exists.

Installing & Updating

Plugin releases are published as .plugin files on GitHub. To install or update:

  1. Download the latest .plugin file from the project’s GitHub Releases.
  2. In Rock, go to Admin Tools → System Settings → Plugins.
  3. Upload the .plugin file. Rock will install the plugin and run any database upgrades automatically on the next page load.
  4. If this is a fresh install, navigate to RockLink › Settings and enter your Planning Center credentials.
