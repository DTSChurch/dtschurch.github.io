Planning Center Sync

One-way sync from Planning Center Online into Rock RMS

Planning Center Sync (com.dts.RockLinkPlanningCenter) pulls People, Groups, Service Teams, and Campuses from Planning Center Online (PCO) into Rock on a configurable schedule. PCO is treated as the source of truth; Rock is the destination. Mappings, conflict policy, and run history are managed through Obsidian admin blocks.

Custom Plugin Rock RMS v17.2+ .NET Framework 4.7.2 Obsidian (Vue 3) PCO REST API

What It Syncs

People

Name, email, phone, address, birthdate, gender, photo, and campus. New people are created with the configured default Connection Status; existing people are matched via EntityMap or first-time matched with FindPerson.

Groups & Service Teams

Rock Group ↔ PCO Group or PCO Service Team mapping, including group members and per-team-position role mapping.

Campuses

Rock Campus ↔ PCO Campus mapping. Used to assign synced people and groups to the correct Rock campus.

What It Does Not Sync (v1)

  • Attendance records — not pulled from PCO.
  • Volunteer schedules — scheduled assignments (e.g. “Person X serving on Date Y”) are not imported.
  • Bi-directional sync — Rock does not push changes back to PCO. The flow is PCO → Rock only.

These are on the future-enhancements list and are not committed for v1.

How It Works

1. Configure credentials

Enter the PCO Application ID and Secret in the Planning Center Settings block. The secret is encrypted at rest. Use Test Connection to validate.

2. Map entities

In the Planning Center Entity Mapping block, link Rock Groups to PCO Groups or Service Teams, map Rock Campuses to PCO Campuses, and (for Service Teams) map PCO team positions to Rock group roles.

3. Scheduled job runs

The SyncPlanningCenter Rock Job runs on its configured schedule (or via Run Sync Now). It walks every active mapping, fetches updated PCO data, resolves to a Rock entity through EntityMap, and creates or updates records per the overwrite policy.

4. Inspect history

The Planning Center Sync History block surfaces every job run with counts (added / updated / skipped / errored), duration, and error details for diagnostics.

Key Configuration Knobs

Setting Effect
Overwrite With Planning Center When on, PCO data overwrites Rock fields on existing linked records. When off, Rock data is preserved and only missing fields get filled in.
Default Connection Status Connection status assigned to brand-new people created by the sync.
Active mappings Only mappings flagged IsActive are processed. Deactivating a mapping pauses sync for that group/team without deleting the link.

Behavior nuance

  • People already linked via EntityMap keep their Rock name on subsequent syncs — Rock is treated as authoritative for the name once linked. Other fields still update per the overwrite policy.
  • First-time linking via FindPerson (name / email / phone match) does still apply PCO names — that is how a brand-new link gets seeded.
  • The EntityMap for a Person links the PCO person to a Rock PersonAlias (not the Person directly), so merges in Rock continue to resolve correctly.

Architecture

Backend — com.dts.PlanningCenterSync

C# / .NET Framework 4.7.2. Contains the PCO REST client, the PlanningCenterSyncService, EF migrations, blocks, and the scheduled job. Compiles to com.dts.RockLinkPlanningCenter.dll.

SyncPlanningCenter Job

Rock Job that orchestrates the sync run, writes a SyncLog record, and aggregates per-mapping counts.

PlanningCenterSyncService

Per-mapping logic: SyncPlanningCenterGroup, SyncServiceTeam, person upsert, member upsert, role mapping.

Migrations

Creates EntityMap, SyncLog, PlanningCenterSettings tables and registers the RockLink admin pages.

Frontend — com.dts.PlanningCenterSync.Obsidian

Vue 3 + TypeScript Obsidian blocks. Built with the Rock Obsidian SDK and shipped to ~/Plugins/com_dts/RockLinkPlanningCenter/.

planningCenterSettings.obs

Credentials, overwrite toggle, default connection status, campus mapping editor, “Run Sync Now”, last run status.

planningCenterEntityMapping.obs

Group / Team / Person mapping editor with role mapping for Service Teams.

planningCenterEntityLink.obs

Inline block for Person Profile and Group Viewer pages — surfaces a deep link into PCO and a per-group Sync Now button.

planningCenterSyncHistory.obs

Grid of past SyncLog rows with counts, duration, status, and error details.

Admin Pages Installed

Migration 004_AddPages registers a top-level RockLink admin section with three child pages.

Page Block(s) Purpose
RockLink › Settings Planning Center Settings API credentials, sync policy, campus mapping, manual run trigger.
RockLink › Groups Planning Center Entity Mapping Manage Rock Group ↔ PCO Group / Service Team mappings.
RockLink › People Planning Center Entity Mapping Inspect Rock Person ↔ PCO Person mappings.
Person Profile Planning Center Entity Link Adds a “View in Planning Center” link on a person’s profile when a mapping exists.
Group Viewer Planning Center Entity Link Adds a deep link and per-group Sync Now on the Group Viewer when a mapping exists.

Release & Install

  • Tagging v* on main triggers release.yml on a Windows runner. The pipeline builds the C# DLL plus the Obsidian bundle and packs a .plugin file via rock-dev-tool, then attaches it to a GitHub Release.
  • Install in Rock via Admin Tools → System Settings → Plugins by uploading the .plugin file. EF migrations run automatically on first request after install.

Future Enhancements

Investigation phase — not committed for v1:

  • Bi-directional sync. Allow Rock to be the source of truth and push changes back to PCO.
  • Attendance sync. Pull attendance records from PCO into Rock.
  • Volunteer schedules. Import scheduled volunteer assignments (e.g. “Axl Rose scheduled for 12/19”).
