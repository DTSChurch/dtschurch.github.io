Custom Lava Filters

Serve System registers custom Lava filters including ServeSignUps, UpcomingAndActiveSignUps, and ServeSignUpDetails. They let template authors query sign-up data from any Lava context. Filters are registered at startup via LavaService.RegisterFilters and are available globally once the plugin is installed.

Global Lava filters Person-based queries ID-based lookup Upcoming schedule preset

UpcomingAndActiveSignUps

Use UpcomingAndActiveSignUps for volunteer-facing schedule lists. It is a named preset over the same shared sign-up query pipeline used by ServeSignUps. 

{% assign signups = CurrentPerson | UpcomingAndActiveSignUps %}

This preset returns non-canceled sign-ups where the opportunity, instance, role, and shift are active. It intentionally ignores instance expiration and uses the scheduled shift date as the cutoff.

A shift scheduled for April 19 remains visible through April 19 at 11:59 PM and stops appearing on April 20. This keeps today's confirmed shift visible even after the sign-up window or instance expiration closes.

ServeSignUps

Returns a list of sign-up detail objects for a given person. Use this as the generic filter when a template needs opportunity, instance, active-state, or date filtering. 

{% assign signups = Person | ServeSignUps %}
{% assign signups = Person | ServeSignUps:42 %}
{% assign signups = Person | ServeSignUps:null,123 %}
{% assign signups = Person | ServeSignUps:null,null,true %}
{% assign signups = Person | ServeSignUps:null,null,null,false %}
{% assign signups = Person | ServeSignUps:null,null,false,true,'2026-03-18' %}

ServeSignUps Parameters

Parameter Type Default Description
input Person — The person to query sign-ups for. Piped in as the Lava input object.
serveOpportunityId int null Filter results to a single serve opportunity.
serveOpportunityInstanceId int null Filter results to a single serve opportunity instance.
activeShiftsOnly bool false When true, only includes sign-ups tied to active shifts.
activeInstanceOnly bool true When true, requires active opportunity and instance and respects the instance end date.
fromDate date string null Only include sign-ups with an occurrence on or after this date.

ServeSignUps remains backward-compatible with its existing positional parameters, but internally it now uses a shared options-based query pipeline. Use UpcomingAndActiveSignUps for the mobile schedule preset.

ServeSignUpDetails

Returns a single sign-up detail object by sign-up ID. Pipe the ID directly into the filter.

{% assign detail = signupId | ServeSignUpDetails %}
Parameter Type Description
input int The sign-up record ID. Returns null if the ID is invalid or not found.

Return Object Reference

Both filters return objects built from BuildSignUpDetails. ServeSignUps returns a list; ServeSignUpDetails returns a single object.

Property Type Description
SignUp ServeOpportunityInstanceSignUp The full sign-up entity.
Shift.Id int Shift record ID.
Shift.Date string Formatted date, e.g. March 19, 2026.
Shift.StartTime string Formatted start time, e.g. 9:00 AM.
Shift.EndTime string Formatted end time, e.g. 10:30 AM.
Shift.StartDateTime DateTime Shift start as a DateTime for date math and custom formatting.
Shift.EndDateTime DateTime Shift end as a DateTime.
Shift.Role ServeOpportunityRole The role entity assigned to the shift.
Shift.RoleName string Role display name.
Shift.Campus Campus Campus entity for the shift, if campus-specific.
Shift.CampusName string Campus display name.
Shift.Schedule ServeOpportunityInstanceSchedule The schedule entity attached to the shift.
Shift.TotalSpots int Total volunteer spots configured for the shift.
Instance ServeOpportunityInstance The instance entity that owns the shift.
InstanceName string Instance display name.
Opportunity ServeOpportunity The top-level opportunity entity.
OpportunityName string Opportunity display name.
Person Person The person who created the sign-up.

Active Sign-Up Checks

UpcomingAndActiveSignUps includes a sign-up only if all of the following are true:

Entity flags are active

The opportunity, instance, role, and shift all have IsActive == true.

Sign-up is not canceled

The sign-up must not have a CanceledDateTime.

Schedule date is current or future

The shift's scheduled date must be today or later.

Instance expiration is ignored

The scheduled shift date, not the instance expiration, controls when the row disappears.

Example Lava Patterns

List upcoming sign-ups for the current person 
{% assign signups = CurrentPerson | UpcomingAndActiveSignUps %}
{% for item in signups %}
  <p>
    <strong>{{ item.OpportunityName }}</strong> &mdash; {{ item.InstanceName }}<br>
    {{ item.Shift.Date }} from {{ item.Shift.StartTime }} to {{ item.Shift.EndTime }}<br>
    Role: {{ item.Shift.RoleName }}
    {% if item.Shift.CampusName != empty %} | Campus: {{ item.Shift.CampusName }}{% endif %}
  </p>
{% endfor %}
Filter by a specific opportunity 
{% assign signups = CurrentPerson | ServeSignUps:42 %}
{% if signups == empty %}
  <p>You have no upcoming sign-ups for this opportunity.</p>
{% else %}
  <ul>
  {% for item in signups %}
    <li>{{ item.Shift.Date }} {{ item.Shift.StartTime }}</li>
  {% endfor %}
  </ul>
{% endif %}
Show all sign-up history (including past) 
{%- comment -%}
  Pass activeInstanceOnly = false and fromDate = a far-past date
  to include inactive-instance and historical sign-ups.
{%- endcomment -%}
{% assign signups = CurrentPerson | ServeSignUps:null,null,false,false,'2020-01-01' %}
{% for item in signups %}
  <p>
    {{ item.OpportunityName }} &mdash; {{ item.Shift.Date }}
    {{ item.Shift.StartTime }} to {{ item.Shift.EndTime }}
  </p>
{% endfor %}
Look up a single sign-up by ID 
{% assign detail = signupId | ServeSignUpDetails %}
{% if detail %}
  <p>{{ detail.Person.FullName }} is signed up for
  <strong>{{ detail.OpportunityName }}</strong>
  on {{ detail.Shift.Date }} at {{ detail.Shift.StartTime }}.</p>
{% else %}
  <p>Sign-up not found.</p>
{% endif %}
