Serve System

Rock RMS volunteer scheduling and sign-up plugin

Serve System extends Rock RMS with structured volunteer opportunity management, recurring scheduling, self-service sign-up, connection follow-up, and targeted coordinator communications across multi-campus serving environments.

Rock RMS v17+ .NET Framework 4.7.2 .NET 8 toolchain Multi-campus scheduling

Who It Serves

Administrators

Create opportunities, manage roles, apply DataView filters, configure content templates, and generate instance schedules.

Volunteers

Browse available opportunities, pick shifts, complete role-specific fields, and receive confirmation with schedule details.

Coordinators

Review sign-ups, receive campus or role-filtered notifications, and optionally move people into Rock Connections for follow-up.

Requirements And Install Paths

Runtime Requirements

  • Rock RMS v17.0 or higher
  • .NET Framework 4.7.2
  • .NET 8 SDK for rock-dev-tool
  • SparkDevNetwork.Rock.DevTool for local plugin development

Production Install

  1. Download the latest plugin package.
  2. Open Rock Shop in Rock RMS.
  3. Install the package and run pending migrations.
  4. Verify Serve pages, blocks, and communication templates are present.

Development Bootstrap

dotnet tool install --global SparkDevNetwork.Rock.DevTool
rock-dev-tool env update
cd serve-system
dotnet restore
dotnet build

Build output copies into the linked Rock environment automatically when the project is configured correctly.

System Model

Serve Opportunity

The root entity that owns role configuration, recipients, templates, and generated instances.

Roles

Per-role sign-up attributes, age rules, optional connection opportunity settings, and campus-aware defaults.

Notification Recipients

People or groups notified after sign-up, optionally filtered by campus and/or role.

Content Templates

Confirmation content selected by role, role category, and campus specificity.

Instances

Scheduled occurrences that own shifts, expirations, and generated sign-up records.

Shifts

Concrete scheduled serve windows, including effective times and optional shift-level expiration overrides.

Sign-Ups

Volunteer registrations tied to roles, shifts, notifications, and optional Rock Connection Requests.

Primary Blocks

Area What It Covers Typical Use
Opportunity List All serve opportunities available to administrators. Used to browse, filter, and open a specific opportunity for review or editing.
Opportunity Detail Opportunity setup, roles, notification recipients, and confirmation templates. Used when reviewing how one ministry or event is configured.
Instance List And Detail Concrete occurrences of an opportunity, including schedules, expirations, and shift-time defaults. Used to review what happens for a specific weekend, event, or campaign instance.
Sign-Up Management Volunteer registrations, status changes, and cancellation behavior. Used to confirm who signed up, what they selected, and how changes are handled afterward.
Role Schedule Configurator Role counts, per-shift expirations, and schedule-specific timing adjustments. Used when teams need to fine-tune staffing and timing at the shift level.
Volunteer Sign-Up The public sign-up experience shown to volunteers and families. Used to review the front-end registration flow and the information a volunteer actually sees.
